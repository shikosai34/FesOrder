# 文化祭モバイルオーダー＆スタンプラリーシステム 使用方法

本書は、本システムをローカル環境で起動し、各ロール（管理者、サークル店員、来場者）がそれぞれの機能を利用するための詳細手順書です。

---

## 1. 事前準備と起動手順

### 1.1. 環境変数の設定
`apps/server/.env` および `apps/web/.env` を設定します。

**サーバー側 (`apps/server/.env`):**
```env
PORT=3001
CORS_ORIGIN=http://localhost:3000
DATABASE_URL=file:./local.db

# 初期の全体システム管理者（event_admin）メールアドレス
INITIAL_SUPER_ADMIN_EMAIL=your-email@example.com

# Better Auth設定
BETTER_AUTH_SECRET=your_very_long_secret_key
BETTER_AUTH_URL=http://localhost:3001
```

**フロントエンド側 (`apps/web/.env`):**
```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

### 1.2. データベースの初期化と反映
データベースのテーブルを作成し、初期化します。

```bash
# スキーマをSQLiteデータベース（apps/server/local.db）に反映
bun db:push
```

### 1.3. 開発サーバーの起動
モノレポ内のすべてのサービス（フロントエンド、サーバー）を一括で起動します。

```bash
bun dev
```

* **フロントエンド (Next.js)**: `http://localhost:3000`
* **バックエンド (Hono)**: `http://localhost:3001`

## 2. アカウントと権限の初期設定

本システムには、イベント全体の管理を行う「イベント管理者」権限が必要です。

### 2.1. イベント管理者（`event_admin`）の自動登録
1. ブラウザで `http://localhost:3000/login`（またはユーザー登録画面 `/register`）を開きます。
2. `apps/server/.env` の `INITIAL_SUPER_ADMIN_EMAIL` に設定したメールアドレスと同じアドレスを使用して、新規ユーザー登録を行います。
3. 登録完了時の最初のリクエストの際、サーバーが自動的にそのユーザーに対して `event_admin`（システム管理者）ロールを割り当てた `membership` を作成します。

---

## 3. イベント管理者（システム管理者）の操作手順

全体管理者は、イベントの立ち上げと、参加サークル（模擬店）の登録を行います。

### 3.1. イベントの新規作成
1. イベント管理者アカウントでログイン後、`http://localhost:3000/admin` にアクセスします。
2. 「イベント新規作成」フォームから、以下の項目を入力して作成します。
   * **イベント名** (例: `第34回 志高祭`)
   * **開催期間** (開始日時 / 終了日時)
   * **説明文**

### 3.2. サークル（模擬店）の作成と代表者一時PIN設定
1. 同画面内の作成したイベントの配下にある「サークル追加」ボタンをクリックします。
2. 以下の情報を入力してサークルを作成します。
   * **サークル名** (例: `たこ焼きサークル`)
   * **代表者一時PINコード** (4〜6桁の数字。例: `1234`。代表者の初回ログイン用)
   * **代表者のメールアドレス**
3. 作成完了後、自動的に代表者のメールアドレス宛てに `circle_manager`（サークル代表者・会計）ロールが付与されます。

## 4. サークル代表者（`circle_manager`）の操作手順

サークル代表者は、自サークルのスタッフの招待やメニューの登録を行います。

### 4.1. サークル管理ダッシュボードへのアクセス
1. サークル代表者のGoogleアカウント（またはメールアドレス）でログインします。
2. `http://localhost:3000/dashboard` を開きます。
3. 自サークルに関連付けられたダッシュボードが表示され、以下のメニューが利用可能になります。
   * **メニュー管理**: 商品とトッピングの追加・編集・削除
   * **在庫管理**: メニューやトッピングの在庫有無・残り個数設定
   * **メンバー管理**: 一般スタッフの招待とロール付与
   * **売上管理**: サークル単体の売上推移確認

### 4.2. メニューとトッピングの登録
1. ダッシュボードから「メニュー管理」を開きます。
2. 「商品を新規追加」から、商品名・価格・写真（アップロード機能対応）・説明・在庫数を登録します。
3. 各商品に対して「トッピング（オプション）」を登録できます。(例: たこ焼きに対して `マヨネーズ (+¥0)`, `大盛り (+¥100)`)

### 4.3. スタッフ（店員）の招待手順
代表者はスタッフ（レジ、厨房など）を招待リンクで登録できます。
1. ダッシュボードから「メンバー管理」を開きます。
2. **スタッフ招待リンクの生成**:
   * 付与するロール（`cashier` (レジ), `kitchen_staff` (厨房), `waiter` (接客), `viewer`）を選択します。
   * 「招待リンクを生成」をクリックし、生成されたリンクをスタッフのSNS等で共有します。
3. **招待されたスタッフの登録手順**:
   * スタッフが共有された招待リンク（`http://localhost:3000/invite/[token]`）にアクセスします。
   * 名前、メールアドレス、および簡易ログイン用の **PIN（4〜6桁の数字）** を入力して「参加する」をクリックします。
   * これにより、スタッフ個人のログインが紐付けられ、次回以降はPINのみで簡単にPOS画面や厨房画面にログイン可能になります。

## 5. サークル一般店員（レジ・厨房）の操作手順

一般スタッフは、店舗内のオペレーション（レジ・調理管理）を行います。

### 5.1. 店舗・スタッフ簡易ログイン
現場の共有端末などで素早くログインするための方法です。
1. `http://localhost:3000/login` にアクセスします（または `/circle-login` にアクセスすると自動でここに転送されます）。
2. 「店舗・スタッフ簡易」タブを選択します。
3. 該当のイベントとサークルを選択し、メールアドレスとPIN（代表者は作成時にシステム管理者が設定した一時PIN、一般スタッフは招待時に自分で登録したPIN）を入力してログインします。

### 5.2. レジ画面（POS）の操作 (`/register`)
レジ担当（`cashier`, `circle_manager`）が使用する画面です。
* **対面注文の受付**: 
  1. 画面上で商品とトッピングを選択してカートに入れます。
  2. 現金または電子決済による支払いを完了させ、「注文確定」します。
  3. 整理番号が記載されたレシート（引換券）をお客さんに渡します。
* **モバイルオーダーの現地決済処理**:
  1. 来場者がスマホから送信した「未決済状態」の注文一覧が画面に表示されます。
  2. お客さんがレジに来たら、整理番号または注文者名で検索し、「決済完了」ボタンを押して支払いを確定します。これで厨房画面に調理依頼が飛びます。

### 5.3. 厨房ビューの操作 (`/backyard`)
厨房担当（`kitchen_staff`, `circle_manager`）が使用する画面です。
1. 決済が完了した注文が、リアルタイムに「未調理」レーンに入ります。
2. **調理開始**: 該当注文をタップして「調理中」にします。
3. **調理完了・呼び出し**: 調理が終わったら「調理完了」にすると、お客さんのスマホ画面に「呼び出し中」と表示されます。
4. **商品渡し**: お客さんが引換券を持ってきたら、商品と引き換えて「受渡済」にして完了します。

## 6. 来場者（一般ユーザー）の操作手順

一般の来場者は、自分のスマートフォンからアカウントを登録し、メニューの注文を行います。

### 6.1. チェックイン（リストバンド登録）とログイン
1. 会場に到着後、デジタルリストバンドのQRコードをスキャンするか、チェックインページにアクセスします。
2. システムを利用するためにはユーザー登録（またはログイン）が必須です。画面の指示に従いアカウントを作成・ログインします。
3. ログイン完了後、自動的にチェックインプロセスにリダイレクトされ、アカウントとリストバンドが紐付けられます。

### 6.2. メニュー閲覧とカート追加
1. `http://localhost:3000/menu`（または各サークルのQRコード）からアクセスします。
2. サークルごとのメニュー一覧から商品を選択し、トッピングを選んでカートに入れます。

### 6.3. モバイルオーダーの送信
1. カート画面で「注文を確定する」ボタンを押します。
2. 注文情報が作成され、**「未決済」の整理番号**が画面に発行されます。

### 6.4. レジでの決済と商品受け取り
1. スマホの注文画面をサークルのレジに見せ、現金などで決済を完了します。
2. 決済されると、スマホ画面が「調理中」に変わります。
3. 調理が完了するとスマホ画面が「呼出中（商品ができました！）」に切り替わります。
4. 画面をスタッフに見せて商品を受け取ります。受け取りが完了すると「受渡済」になります。
